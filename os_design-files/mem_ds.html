File Table



The File Table stores the information about all the files that are currently open while the OS is running. It consists of MAX_OPENFILE_NUM (= 32 in the present eXpOS version) entries. Thus, there can be at most MAX_OPENFILE_NUM = 32 open files in the system at any time.

The Open system call creates an entry in the File table when a process opens a file that is not opened by any process in execution. If the file is opened again by some other process (or the same process), the Open system call updates the same file table entry. The first field (Inode Index) of the File table entry for a file is set to the index of the Inode Table entry for the file. The second field (File Open Count) stores the number of open instances of the file and is incremented/decremented when a process opens/closes the file. The third field (Locking PID) is set to the PID of the process that has locked the file using the Flock system call. If the file is not locked by any process, this field contains -1.

If the file open count reaches zero (that is if the last open instance has been closed) then, all fields of the corresponding file table entry is invalidated by setting to -1.

Each entry is of size 4 words of which the last one is left unused.

Every entry of the File Table has the following format :

INODE INDEX FILE OPEN COUNT LOCKING PID Unused
  • INODE INDEX (1 word) - specifies the index of the entry for the file in the Inode table.
  • FILE OPEN COUNT (1 word) - specifies the number of open instances of the file
  • LOCKING PID (1 word)- specifies the PID of the process which has curently locked the file.
  • Unused (1 word)



Semaphore Table



It contains details about all the semaphores used by the processes. For every semaphore entry in the per-process resource table, there is a corresponding entry in the semaphore table. Number of entries in the ST is defined by the constant MAX_SEM_COUNT.

Every entry of the Semaphore Table has the following format :

LOCKING PID PROCESS COUNT
  • LOCKING PID - specifies the PID of the process which has locked the semaphore
  • PROCESS COUNT - specifies the number of processes waiting for semaphore



Disk Status Table



It keeps track of the load and store operations.

Every entry of the Disk Status Table has the following format:

LOAD/STORE BIT PAGE NUMBER BLOCK NUMBER PID
  • LOAD/STORE BIT - specifies whether the operation being done on the device is a load or store.
  • PAGE NUMBER - specifies the page number of the page, which is being stored onto the disk or, to which a block from disk is being loaded.
  • BLOCK NUMBER - specifies the block number of the disk block being loaded into memory or to which a page is being stored.
  • PID - specifies the PID of the process who invoked the device.



Buffer Table



To minimize the number of load and store operations, eXpOS provides a buffer cache in memory which would temporarily store disk blocks. Each disk block is mapped to a unique buffer. The disk blocks will be stored back to the disk when some other blocks replace it. Only modified blocks are written back to disk.

It keeps the information about disk blocks present in the buffer. There are MAX_BUFFER number of buffers available so the BUFFER TABLE will have that many entries.

Every entry of the Buffer Table has the following format:

BLOCK NUMBER DIRTY BIT LOCKING PID
  • BLOCK NUMBER- specifies block number of the disk block stored in the buffer.
  • DIRTY BIT - specifies whether the block stored in the buffer has been modified.
  • LOCKING PID - specifies the PID of the process which has currently locked the buffer.



System Status Table



It keeps the information about no: of free pages in memory, no: of processes waiting for memory pages and also no: of processes in swapped state.

The System Status Table has the following format:

MEM_FREE_COUNT WAIT_MEM_COUNT SWAPPED_COUNT
  • MEM_FREE_COUNT- specifies the number of free pages available in memory.
  • WAIT_MEM_COUNT - specifies the number of processes waiting for memory.
  • SWAPPED_COUNT - specifies the number of processes in SWAPPED state.



Memory Free List



The Memory free list is a data structure used for keeping track of used and unused pages in the memory. It consists of MAX_MEM_PAGE entries and each entry is of size one word. Thus, the total size of the free list is MAX_MEM_PAGE words.

Each entry of the free list contains a value of either 0 indicating whether the corresponding page in the memory is free or number indicating the number of processes that share the page.
