Open System Call



Arguments: Filename (String)

Return Value:

PerprocessIndex Success

Description: The Open operation takes as input the Filename of the file to be opened. It checks for the validity of the file type, creates an entry for the file in the Per Process Resource Table of the process and also in the File Table.It returns from the system call with the perprocessindex of the file.

Algorithm:

If file is not present in inode table

then Return -1

If file type is not DATA or ROOT

then Return -1

Store the Index corresponding to the Inode Table entry in InodeIndexAllocate entry in Per Process Resource Table

1. Find the PID of the currently running process from the Process Table. This will be the PID of unique process in the state (RUNNING, _).

2. Search for a free entry in the Per-Process Resource table of the process.

if there is no free entry

then Return -1

else

Store the index of the Per-Process Resource table entry in PerProcessIndex

Allocate entry in File Table

1. Check if file is already open by some process

Search for an entry in the File Table entry with Inode Index field as InodeIndex

if an entry is found

then Store the index of the File Table entry in FTIndex

else

Search the File Table for a free entry.

If there are no free entries

then Return -1

else

Store the index of that File Table entry in FTIndex.

Update the Per-Process Resource Table entry.

Set the first field (Index of File Table) of this entry FTIndex.

Set LSEEK Position field to 0.

Update the File Table entry.

Set the Inode Index field of the File Table entry FTIndex, with InodeIndex.

Increment the File Open Count field in the File Table.

Set the Locking PID field to -1.

Return from System Call with PerprocessIndex, indicating success
