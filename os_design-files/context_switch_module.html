Module 5: Context Switch Module

Context switch module is loaded as module 5 into the XSM disk.

Context switch module provides the Switch_Context() functionality. The next process to be scheduled is selected using round robin scheduling algorithm.



Switch_Context()



Arguments: NIL

Algorithm:

Obtain the PID current process from System Staus Table.

Save the context of the current process into its Process Table entry.

Obtain the process table entry as PROCESS_TABLE + currentPID * 16.

Save SP % 512 in the kernel stack pointer field of the Process Table.

Save PTBR and PTLR into the corresponding fields in the Process Table.

Find the next process to schedule using Round Robin scheduling algorithm.

Iterate through the process table entries to find a process in READY or CREATED state.

If no such process can be found, select idle as the new process to be scheduled.

Restore the context of the new process from its Process Table entry.

Obtain User Area Page number and kernel stack pointer value from Process Table entry of the new process.

Set SP as User Area Page number + Kernel Stack pointer.

Restore PTBR and PTLR from the corresponding fields in the Process Table entry.

Set the PID of the new process in the current PID field of System Status Table.

Check if the new process is in CREATED state.

Set SP as User SP from its Process Table entry.

Set its state as RUNNING.

Ireturn.

Set the state of new process as RUNNING.

Return.
