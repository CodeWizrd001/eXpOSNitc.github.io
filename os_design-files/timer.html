Timer Interrupt Handler



The hardware requirement specification for eXpOS assumes that the machine is equipped with a timer device that sends periodic hardware interrupts. The timer interrupt handler internally invokes the eXpOS scheduler module .


Algorithm:

Save the SP register to User SP field of Process Table entry of the current process.

Set the value of SP to the beginning of the kernel stack of the current process. (This will be the beginning of the User Area page of the process).

Backup the context of the current process using BACKUP instruction.

Change the state of the current process in its Process Table entry from RUNNING to READY.

Invoke the context switch module using the call MOD_5 instruction.

// swap module will be invoked whenever necessary

Restore the context of the new process using RESTORE instruction.

Set SP as the user SP saved in the Process Table entry of the new process.

Set the MODE_FLAG in the process table entry to 0.

Ireturn.
