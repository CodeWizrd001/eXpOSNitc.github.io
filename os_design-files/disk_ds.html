Inode Table



Inode table has an entry for each file present in the disk. It consists of MAX_FILE_NUM entries. Thus there can be a maximum of MAX_FILE_NUM files.

The entry of an Inode table has the following format:

FILE TYPE FILE NAME FILE SIZE DATA BLOCK 1 DATA BLOCK 2 DATA BLOCK ... DATA BLOCK N
  • FILE TYPE - specifies the type of the given file (ROOT, DATA or EXEC).
  • FILE NAME - Name of the file
  • FILE SIZE - Size of the file
  • DATA BLOCK 1 to N - each DATA BLOCK column stores the block number of a data block of the file.


Disk Free List



The Disk Free List consists of DISK_SIZE entries. Each entry is of size one word and thus, the size of the free list is 1 block or DISK_SIZE words. For each block in the disk there is an entry in the Disk Free List which contains a value of either 0 or 1 indicating whether the corresponding block in the disk is free or used, respectively.

When the system starts up, the OS startup code loads the Disk Free List to memory. It is stored back when the system halts or a Shutdown system call is called.
