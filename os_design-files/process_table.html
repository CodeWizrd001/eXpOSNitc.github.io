Process Table



The Process Table (PT) contains an entry for each process present in the system. The entry is created when the process is created by a Fork system call. Each entry contains several fields that stores all the information pertaining to a single process. The maximum number of entries in PT i.e., maximum number of processes allowed to exist at a single point of time in eXpOS, is MAX_PROC_NUM. In the current version of eXpOS, MAX_PROC_NUM = 32.

Each entry in the Process Table has a constant size, defined by the constant PT_ENTRY_SIZE. Any entry of PT has the following fields:

TICK PID PPID STATE PTBR PTLR PER-PROCESS RESOURCE TABLE INODE INDEX iSTREAM MACHINE STATE KERNEL STACK POINTER Unused
  • TICK (1 word)- keeps track of how long the process was in memory.
  • PID (1 word) - process descriptor, a number that is unique to each process.
  • PPID (1 word) - process descriptor of the parent process.
  • STATE (2 words) - a two tuple that describes the current state of the process.
  • MACHINE STATE - pointer to a structure that gives the machine state when the process was last executed.
  • PTBR (1 word)- Page Table Base Register stores the starting address of the page table of a process.
  • PTLR (1 word)- Page Table Length Register stores the number of entries in the page table of a process and determines the size of the virtual address space of the process. (size of the virtual address space is page table length register entry times page size).
  • PER-PROCESS RESOURCE TABLE - pointer to the Per-Process Resource Table that contains information about the files opened by the process as well as semaphores used by the process.
  • INODE INDEX (1 word)- a reference to the Inode entry of the executable file
  • KERNEL STACK POINTER (1 word) - pointer to the process's kernel stack. A process uses its kernel stack to save the return address when it decides to voluntarily schedule out itself inside a blocking system call.

Invalid entries are represented by -1.



STATE

The tuple can take the following values

  • (RUNNING,___) – The process is in execution.
  • (READY,___) – The process is ready to be scheduled.
  • (WAIT_PROCESS,WAIT_PID) – The process is waiting for a signal from another process whose PID is WAIT_PID.
  • (WAIT_FILE, FTENTRY) - The process is waiting for accessing a file whose file table entry index is FTENTRY.
  • (WAIT_DISK,___) - The process is blocked because of one of the following reasons: a) It is waiting for disk to complete a disk-memory transfer operation it had initiated OR b) It wants to execute a disk transfer, but the disk is busy, handling a disk-memory transfer request issued by some other process.
  • (WAIT_SEMAPHORE,SEMID) - The process is waiting for a semaphore (whose descriptor is SEMID) that was locked by some other process.
  • (WAIT_MEM, ___) – The process is blocked due to unavailability of memory pages.
  • (SWAPPED,___) – The stack page of the process has been swapped out to disk
  • (WAIT_BUFFER, BUFFERID) – The process is waiting for disk buffer of index BUFFERID to be unlocked.


MACHINE STATE

This part is machine dependent. The OS scheduler can schedule out a process in execution and save its machine state here. Later, the scheduler can restore the machine state to start execution from where it was stopped. The initial values of this structure are fixed by the OS loader (exec system call) and the values get updated whenever the process is preempted.

The MACHINE STATE consists of USER CONTEXT and SYSTEM CONTEXT. Each would contain the following information.

SP BP IP REGISTERS
  • SP (1 word)- Stack pointer pointing to top of the stack.
  • BP (1 word) - Base pointer pointing to the base of the stack.
  • IP (1 word) - Instruction pointer pointing to the next instruction to be executed.
  • REGISTERS - Machine registers.

USER CONTEXT is updated when a process is preempted when it is in user mode. SYSTEM CONTEXT is updated when a there is a context switch from kernel mode to kernel mode.



PER-PROCESS RESOURCE TABLE

For every file opened by the process, it stores the index of the File Table entry for the file, and the LSEEK position of the open instance of a file. The LSEEK position indicates the location in the file where a word is read from or written to corresponding to the open instance of the file.

Similarly, for every semaphore used by the process, the per-process resource table stores the index of Semaphore Table entry.

Each entry is of size 2 words. In this versions of eXpOS, number of entries in the Per-Process Resource Table is 8. Thus per-process resource table has a size of 16 words.

The per-process resource table entry has the following format.

Index of File Table/ Semaphore Table entry (1 word) LSEEK (1 word)

The index of the per-process resource table entry for each file is known as the file descriptor.
