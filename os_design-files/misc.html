Timer Interrupt



Description:

The hardware requirement specification for eXpOS assumes that the machine is equipped with a timer device that sends periodic hardware interrupts. The OS scheduler is invoked by the hardware timer interrupt handler. eXpOS specification suggest that a co-operative multitasking round robin scheduling is employed. Hence a process may go to sleep inside a system call when:

  1. The resource which the process is trying to access (like a file or a semaphore) is locked by another process (or even internally locked by another OS system call in concurrent execution).
  2. There is a disk/terminal access in a system call which has blocked the process. When the OS blocks a process, it continues scheduling the remaining processes in round robin fashion and then wakes up the blocked process when the device sends an interrupt.

Algorithm:

Save the context of current process

1. Find the PID of the currently running process from the Process Table. This will be the PID of unique process in the state (RUNNING, _)

2. Save the machine state to the location pointed to by Machine State Pointer.

3. Change the state of the current process from RUNNING to READY by setting the STATE field of the Process Table entry to (READY, _).

Find the next process to be scheduled and the senior most swapped process

Scan the Process Table in Round Robin fashion to find the next proceess in ready state (that is, state field has value (READY, _)) and also the process with highest value for TICK.

While scanning increment the TICK of all the process.

If free memory pages present

Check the MEM_FREE_COUNT in System Status Table.

If there are sleeping processes that requires memory pages

Check the WAIT_MEM_COUNT in System Status Table.

Wake up all processes that requires memory pages.

Change the state from (WAIT_MEM,_) to (READY,_).

Reset the WAIT_MEM_COUNT to 0.

Check the SWAPPED_COUNT in System Status Table.

Find a free page in memory.

Set the state of the process to ready.

Decrement the SWAPPED_COUNT and MEM_FREE_COUNT.

If there are sleeping processes that requires memory pages

Check the WAIT_MEM_COUNT in System Status Table.

Use second chance algorithm to find an unreferenced page.

If the unreferenced page is a code page

Using pointer to Inode Table, find the corresponding code block.

Set the page table entry of the selected page to the code block number.

Set auxiliary information to unreferenced and invalid.

If the unreferenced page is a stack or heap page.

Find a free block in swap area.

Store the selected page in the swap block.

Set the page table entry to the swap block number and auxiliary information to unreferenced and invalid.
