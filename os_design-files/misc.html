Timer Interrupt



The hardware requirement specification for eXpOS assumes that the machine is equipped with a timer device that sends periodic hardware interrupts. The Timer Interrupt routine handler implements scheduling in eXpOS. The eXpOS specifies that a co-operative multitasking round robin scheduling is employed. The timer interrupt routine in eXpOS does the following tasks:

  1. Schedules the next ready process for execution.
  2. Swaps in (or out) processes from (or to) the disk.
  3. If unused memory pages are present in the system, timer interrupt routine wakes up those processes which were blocked due to lack of memory.

Algorithm:

Save the context of current process

1. Find the PID of the currently running process from the Process Table. This will be the PID of unique process in the state (RUNNING, _).

2. Save the machine state to the location pointed to by Machine State Pointer.

3. Change the state of the current process from RUNNING to READY by setting the STATE field of the Process Table entry to (READY, _).

Increment TICK   /* Increment the TICK of all the processes. TICK is given in Process Table. */

If free memory pages present     /* Check the MEM_FREE_COUNT in System Status Table. */

If there are sleeping processes that requires memory pages     /* Check the WAIT_MEM_COUNT in System Status Table. */

Wake up all processes that requires memory pages.

1. Change the state from (WAIT_MEM,_) to (READY,_).

2. Reset the WAIT_MEM_COUNT to 0.

If disk is free     /* Check the Disk Status Table. */

If there are swapped processes     /* Check the SWAPPED_COUNT in System Status Table. */

For the senior most swapped process, find the block which has to be swapped in.

1. Process with highest value for TICK among the swapped process is the seniormost process.

2. Using the SP value in the Machine State, find the stack page to be swapped in.

Find a free page in memory by checking the Memory Free List.

Load the block (in which the stack page is stored) from disk to the free page in memory.

In the page table entry corresponding to the stack page, set the page number to the number of the free page.

Set the auxiliary information of the page table entry to referenced and valid.

Set the state of the process to (READY, _).

Reset the TICK count of the process to 0.

Decrement the SWAPPED_COUNT and MEM_FREE_COUNT.

Find the PID of the process that owns the page and locate its page table.  /*Check the Process Table*/

If the unreferenced page is a code page(See Memory layout)   /* since code pages are not modified, they need not be stored back to the disk */

Using pointer to Inode Table, find the corresponding code block number
