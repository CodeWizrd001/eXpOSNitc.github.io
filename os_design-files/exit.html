Exit system call



Arguments: None

Return Value: None

Description: Exit system call terminates the execution of the process which invoked it and destroys its memory address space. The calling application ceases to exist after the system call and hence the system call never returns.


Algorithm:

If no more processes to schedule     /* Check the Process Table for any process in READY state. */

shutdown the machine. /* use Shutdown system call */

else

Find the PID of the next ready process from the Process Table.

Close all files opened by the current process.     /* Follow the procedures in Close system call */

Release all the semaphores used by the current process.     /* Follow the procedures in Semrelease system call */

Memory pages of the current process are freed.

1. In the Memory Free List, decrement the value corresponding to the pages.

2. If the value becomes 0, then increment MEM_FREE_COUNT.

Invalidate the page table entries of the current process

Wake up all processes waiting for the current process.

1. In the Process Table, find the processes in WAIT_PROCESS state, having current PID as WAIT_PID.

2. Change the state of the processes to READY.

Schedule the next ready process.

Return to user mode.
