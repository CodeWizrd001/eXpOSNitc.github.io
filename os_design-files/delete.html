Delete System Call



Arguments: Filename (String)

Return Value:

0 Success
-1 File not found or File is not a DATA file
-2 File is open

Description : The Delete operation takes as input a filename and deletes it. It returns with an error if any instance of the file is open in the system or if the file is not a DATA file. Otherwise, it deletes the root entry for the file name, invalidates the Inode Table entry for the file, releases the disk blocks allocated to the file and returns 0.

The data structures modified are Inode Table, Root file and Disk Free List.

The mode flag in the Process Table has to be set to Kernel mode when the process enters the system call and reset before exiting from the system call.

Algorithm:

If file is not present in Inode Table or if it is not a DATA file, return -1.

Find the index of the file in the Inode Table.

If Open File Table entry exists with the same index as found above, return -2.   /* File is open, cannot be deleted */

Update Inode Table

For each disk block of the file, decrement its entry in the memory copy of Disk Free List.    /* release the disk blocks */

Invalidate (set to -1) the Inode Table entry.

Update the Root file by invalidating the root entry for the file.

Return from system call with 0.    /* indicating success */
