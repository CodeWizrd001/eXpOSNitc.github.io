Delete System Call



Arguments: Filename (String)

Return Value:

0 Success
-1 File not found or File is not a DATA file
-2 File is open

Description: The Delete operation takes as input a filename and deletes it. It returns with an error if any instance of the file is open in the system or if the file is not a DATA file. Otherwise, it deletes the root entry for the file name, invalidates the Inode Table entry for the file and returns 0. Note that the file name must be a character string and must not be “root”.

Algorithm:

If file is not present in Inode Table or if it is not a DATA file, return -1

Find the index of the file in the Inode Table

If File Table entry exists with the same index as found above, return -1

Update Inode Table

1. Find the Inode Table Entry corresponding to InodeIndex

2. For each DATABLOCK of the file invalidate its corresponding entry (set to -1) in the Memory copy of Disk Free List.

3. Invalidate (set to -1) the Inode Table entry.

Update the root file by removing the root entry for the file. Decrement the file size of root file.

Return from System Call with 0, indicating success
