Fork System Call



Arguments: None

Return Value:

PID (Integer) Success, the return value to the parent is the process descriptor(PID) of the child process.
0 Success, the return value to the child.
-1 Failure, Number of processes has reached the maximum limit. Returns to the parent

Description: Replicates the process invoking the system call. The heap, code and library regions of the parent are shared by the child. A new stack is allocated to the child and the parent's stack is copied into the child's stack.

When a process executes the Fork system call, the child process shares with the parent all the file and semaphore descriptors previously acquired by the parent. Semaphore/file descriptors acquired subsequent to the fork operation by either the child or the parent will be exclusive to the respective process and will not be shared.


Algorithm:

If no free entry in Process Table

return -1

else

Find the index of free entry in Process Table.    /* This entry is assigned to the child process */

Set the PID field of the free Process Table entry to the child's PID.

Set the PPID field of child process to the current PID.

Count the number of stack pages of parent

while equal number of free pages are not present in memory do    /* Check the MEM_FREE_COUNT in System Status Table. */

Change the state of the current process to WAIT_MEM.

Call scheduler.

Allocate stack pages for child process.

1. Find a free memory page for each stack page of the parent.

2. Update the Memory Free List by incrementing the entry for the selected page.

3. Decrement MEM_FREE_COUNT.

4. In the page table entry of the child's stack, set the page number to the number of the selected page.

5. Auxiliary information is copied from the page table of the parent process.

Copy the parent's stack pages to the child's stack pages.

Copy the page table entries, except stack entries, of parent to the page table of child.

Copy the parent's Machine State and Per-Process Resource Table to that of the child.

Copy the Inode Index in Process Table entry of the parent to child's Inode Index field.

For every open file of the parent, increment the File Open Count in the File Table.

For every semaphore acquired by the parent, increment Process Count in the Semaphore Table.

Set state of child process to READY.

Return 0 to the child process and PID of the child process to the parent process.
