System Call Interface



Application programmers interact with the Operating System using the system calls. System calls are stored in the disk and are loaded into memory when the OS is loaded by the bootstrap loader. When a process invokes a system call, the process is interrupted and control goes to the corresponding interrupt service routine of the kernel, resulting in a switch from user mode to kernel mode. Once the system call is carried out, the control goes back to the application program, with a switch back to the user mode.

The system calls of eXpOS are classified into file system calls, process system calls and system calls for access control and synchronization.



File System Calls

Create

Arguments: String indicating Filename

Return Value:

0 Success
-1 No space for file

Description: The Create operation takes as input a filename and creates an empty file by that name. It also creates the root entry for the file and makes sure that every file has a unique name. Note that the file name must be a character string and must not be named “root”.

Delete

Arguments: String indicating Filename

Return Value:

0 Success
-1 File not found
-2 File is open

Description: Delete removes the file from the file system and removes its root entry. A file that is currently opened by any application cannot be deleted. Root file also cannot be deleted.

Open

Arguments: String indicating Filename

Return Value:

File Descriptor Success, the return value is the file descriptor for the opened file.
-1 File not found or file is not a data or root file
-2 Process has reached its limit of resources
-3 System has reached its limit of open files

Description: For a process to read/write a file, it must first open the file. Only data and root files can be opened. The Open operation returns a file descriptor. The file descriptor conceptually refers to the location of the file which will be referred to by other file operations. The file pointer initially points to the first word in the file when Open system call is completed. An application can open the same file several times and each time a different descriptor will be returned by the Open operation.

Close

Arguments: Integer indicating File Descriptor

Return Value:

0 Success
-1 File Descriptor given is invalid

Description: After all the operations are done, the user closes the file using Close system call.

Read

Arguments: Integer indicating FileDescriptor and a String Buffer to which the word is read

Return Value:

0 Success
-1 File Descriptor given is invalid
-2 File pointer has reached the end of file

Description: Read reads one word into a string/integer variable from the position pointed by file pointer in the file specified by the input file descriptor. After each read is performed, the file pointer advances to the next word in the file.

Write

Arguments: Integer indicating FileDescriptor and a String/Integer indicating the Word to be written

Return Value:

0 Success
-1 File Descriptor given is invalid
-2 No disk space

Description: Write writes one word from a string/integer variable provided by the user, to the position pointed by input file pointer. After each write is performed, the file pointer advances to the next word in the file.

Seek

Arguments: Integer indicating File Descriptor and an integer Offset specifying the number of positions by which the file pointer has to be shifted

Return Value:

0 Success
-1 File Descriptor given is invalid
-2 Offset value moves the file pointer to a position outside the file

Description: If user wants to change the file pointer without reading/writing, Seek system call is used. The Seek operation allows the application program to change the location pointed to by the file pointer from the current position to a different position in the file so that next Read/Write is performed from the new position. An Offset of 0 will reset the pointer to the start of the file. A negative Offset will move the pointer backwards.



Process System Calls

Fork

Arguments: None

Return Value:

PID Success, the return value to the parent is the process descriptor(PID) of the child process.
0 Success, the return value to the child.
-1 Number of process has reached its maximum

Description: Replicates the process invoking the system call. The heap, code and library regions of the parent are shared by the child. A new stack is allocated to the child and the data in the parent stack is copied into the child stack.

When a process executes the Fork system call, the child process shares with the parent all the file handles and semaphores previously opened by the parent. Note that semaphores and the files handles acquired subsequent to the fork operation by either the child or the parent will be exclusive to the respective process and will not be shared.

Exec

Arguments: String indicating File Name of the executable file (which must be of XEXE format)

Return Value:

-1 File not found or file is of invalid type

Description: Exec loads the executable program into the memory address space of the calling process, overlaying the calling application with the newly loaded program. After the Fork system call, if either the parent or the child process loads another program into its virtual address space using the exec system call, then the shared heap is detached from that process and the surviving process will have the heap intact. A successful Exec operation results in extinction of the calling application and never returns to the caller application. For the new process, no heap pages are allocated. All open instances of file and semaphores of parent process is closed.

Exit

Arguments: None

Return Value:

-1 Failure

Description: Exit system call terminates the execution of the process which invoked it and removes it from the memory. It schedules the next ready process and starts executing it. When there is no other ready process to run, it halts the machine.

Getpid

Arguments: None

Return Value:

Process Identifier Success
-1 Failure

Description: Returns the process identifier of the invoking process.

Getppid

Arguments: None

Return Value:

Parent Process Identifier Success
-1 Failure

Description: Returns the process Identifier of the parent of the invoking process.



System Calls for access control and synchronization

Wait

Arguments: Integer indicating Process Identifier of the process for which the current process has to wait.

Return Value:

0 Success
-1 Given process identifier is invalid or it is the pid of the invoking process.

Description: The current process is blocked till the process with PID given as argument executes a Signal system call or exits. It is to be noted that error will occur if a process is found to wait for itself.

Signal

Arguments: None

Return Value: 0

0 Success
-1 Failure

Description: All processes waiting for the signaling process are resumed.

FLock

Arguments: String indicating File Descriptor

Return Value:

0 Success
-1 File Descriptor is invalid

Description: To lock a file so that other applications running concurrently are not permitted to access the file till the file is unlocked. If the file is found to be locked by some other process, the current process waits till the file is free and then locks it.

FunLock

Arguments: Integer indicating FileDescriptor

Return Value:

0 Success
-1 File Descriptor is invalid
-2 File is locked by some other process

Description: Funlock operation allow an application program to unlock a file the application had locked earlier so that other applications are no longer restricted from accessing the file. Note that it is a SERIOUS implementation error if the file is found to be locked by some other process.

Semget

Argument: None

Return Value :

Integer indicating the SEMID Success, returns the semaphore descriptor(SEMID)
-1 Process has reached its limit of resources
-2 Number of semaphores has reached its maximum

Description: Semget does not take any input and gives the semaphore handle as the output. eXpOS has a fixed number of semaphores. An error code is produced if there are no available semaphores in the system.

This system call, if successful, is used to obtain a semaphore for a resource. The semaphore thus acquired can later become shared in case of forking.

Semrelease

Arguments: Integer indicating Semaphore Descriptor

Return Value:

0 Success
-1 Semaphore Descriptor is invalid

Description: This system call is used to free the semaphore so that other processes can use it.

SemLock

Arguments: Integer indicating Semaphore Descriptor

Return Value:

0 Success
-1 Semaphore Descriptor is invalid

Description: This system call is used to lock the semaphore. If the semaphore is already locked, the process goes to sleep and wakes up only when the semaphore is reset.Otherwise, it locks the semaphore and continues execution.

SemunLock

Arguments: Semaphore Descriptor

Return Value:

0 Success
-1 Semaphore Descriptor is invalid

Description: This system call is used to unlock the semaphore. It will reset the semaphore, waking up all processes which went to sleep trying to lock the semaphore.
