System Call Interface



Application programmers interact with the Operating System using the system calls. System calls are stored in the disk and are loaded into memory when the OS is loaded by the bootstrap loader. When a process invokes a system call, the process is interrupted and control goes to the corresponding interrupt service routine of the kernel, resulting in a switch from user mode to kernel mode. Once the system call is carried out, the control goes back to the application program, with a switch back to the user mode.

The system calls of eXpOS are classified into file system calls, process system calls and system calls for access control and synchronization.



File System Calls

Create

Arguments: String indicating Filename

Return Value:

0 Success
-1 No space for file

Description: Create allows application programs to create a new data file with the Filename specified in the argument. It returns successfully when the file has been created or if a file, with the given Filename, already exists.

Delete

Arguments: String indicating Filename

Return Value:

0 Success
-1 File not found
-2 File is open

Description: Delete is used to delete a file from the eXpFS disk. It returns successfully when the file has been deleted from the disk. This operation fails if the file has any open instances or if file does not exist in the disk.

Open

Arguments: String indicating Filename

Return Value:

File Descriptor Success, the return value is the file descriptor for the opened file.
-1 File not found or file is not a data or root file
-2 Process has reached its limit of resources
-3 System has reached its limit of open files

Description: Open is used to open a file present in the eXpFS disk. When it returns successfully, the process will have an open instance of the file which can be accessed using the File Descriptor. Only data and root files can be opened.

Close

Arguments: Integer indicating File Descriptor

Return Value:

0 Success
-1 File Descriptor given is invalid

Description: Close is used to close a file.

Read

Arguments: Integer indicating FileDescriptor and a String Buffer to which the word is read

Return Value:

0 Success
-1 File Descriptor given is invalid
-2 File pointer has reached the end of file

Description: Read reads a word from a data file and places it in the Buffer passed as argument.

Write

Arguments: Integer indicating FileDescriptor and a String indicating the Word to be written

Return Value:

0 Success
-1 File Descriptor given is invalid
-2 No disk space

Description: Write writes the Word given into the file at the position of the file pointer.

Seek

Arguments: Integer indicating File Descriptor and an integer Offset specifying the number of positions by which the file pointer has to be shifted

Return Value:

0 Success
-1 File Descriptor given is invalid
-2 Offset value moves the file pointer to a position outside the file

Description:Seek allows the application program to change the location pointed to by the file pointer so that next Read/Write is performed from the new position. An Offset of 0 will reset the pointer to the start of the file. A negative Offset will move the pointer backwards.



Process System Calls

Fork

Arguments: None

Return Value:

PID Success, the return value to the parent is the process descriptor(PID) of the child process.
0 Success, the return value to the child.
-1 Number of process has reached its maximum

Description: Replicates the process invoking the system call. The heap, code and library regions of the parent are shared by the child. A new stack is allocated to the child and the data in the parent stack is copied into the child stack. All open file handles and semaphores are shared between the parent and the child.

Exec

Arguments: String indicating File Name of the executable file

Return Value:

-1 File not found or file is of invalid type

Description: Loads and runs a new process in the space of a currently running process. The current process is overwritten by new process i.e. code and stack regions of the current process is used by the new process. The heap pages of the current process will be freed, if they are not shared with any other process. For the new process, no heap pages are allocated .All open instances of file and semaphores of parent process is closed.

Exit

Arguments: None

Return Value:

-1 Failure

Description: Destroys the process invoking the call. If successful, it does not return any value. If the current process is the only process that has not terminated, system is made to shutdown.

Getpid

Arguments: None

Return Value:

Process Identifier Success
-1 Failure

Description: Returns the process identifier of the invoking process.

Getppid

Arguments: None

Return Value:

Parent Process Identifier Success
-1 Failure

Description: Returns the process Identifier of the parent of the invoking process.



System Calls for access control and synchronization

Wait

Arguments: Integer indicating Process Identifier of the process for which the current process has to wait.

Return Value:

0 Success
-1 Given process identifier is invalid or it is the pid of the invoking process.

Description: The current process is blocked till the process with PID given as argument executes a Signal system call or exits. It is to be noted that error will occur if a process is found to wait for itself.

Signal

Arguments: None

Return Value: 0

0 Success
-1 Failure

Description: All processes waiting for the signaling process are resumed.

FLock

Arguments: String indicating File Descriptor

Return Value:

0 Success
-1 File Descriptor is invalid

Description: To lock a file so that other applications running concurrently are not permitted to access the file till the file is unlocked. If the file is found to be locked by some other process, the current process waits till the file is free and then locks it.

FunLock

Arguments: Integer indicating FileDescriptor

Return Value:

0 Success
-1 File Descriptor is invalid
-2 File is locked by some other process

Description: Funlock operation allow an application program to unlock a file the application had locked earlier so that other applications are no longer restricted from accessing the file. Note that it is a SERIOUS implementation error if the file is found to be locked by some other process.

Semget

Argument: None

Return Value :

Integer indicating the SEMID Success, returns the semaphore descriptor(SEMID)
-1 Process has reached its limit of resources
-2 Number of semaphores has reached its maximum

Description: This system call is used to obtain a semaphore for a resource. The semaphore thus acquired can later become shared in case of forking.

Semrelease

Arguments: Integer indicating Semaphore Descriptor

Return Value:

0 Success
-1 Semaphore Descriptor is invalid

Description: This system call is used to free the semaphore so that other processes can use it.

SemLock

Arguments: Integer indicating Semaphore Descriptor

Return Value:

0 Success
-1 Semaphore Descriptor is invalid

Description: This system call is used to lock the semaphore. If the semaphore is already locked, the process goes to sleep and wakes up only when the semaphore is reset.Otherwise, it locks the semaphore and continues execution.

SemunLock

Arguments: Semaphore Descriptor

Return Value:

0 Success
-1 Semaphore Descriptor is invalid

Description: This system call is used to unlock the semaphore. It will reset the semaphore, waking up all processes which went to sleep trying to lock the semaphore.
