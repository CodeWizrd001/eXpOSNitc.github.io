Multi-user Extension to eXpOS



This document specifies the extended features provided by eXpOS to support multiple users.

The extended eXpOS allows a maximum of MAX_USER_NUM users. Each user is alloted a unique userid. Every process is assigned the userid, corresponding to the user executing the process. Two special processes hand created by the kernel at boot-足time are given the special userid 足0. These processes are the idle process and the login process and are called the kernel processes. At the end of system bootstrap, the login process is the first process scheduled for execution. The present version of eXpOS sets MAX_USER_NUM to 16.

The login process waits invoking the (terminal) read system call for a username and password to be input from the console. Both username and password are strings. The login process executes the Login system call with the username and the password as the arguments. The login system call verifies the user and creates a new shell process. The shell is assigned the userid of the logged in user(by the Login system call). The original login process waits (using the Wait system call) for the shell to exit, to login the next user. Once a shell process is created with one user logged in, all descendent processes of the shell inherit the userid of the shell. Note that Login is the only system call that can set the userid for a process. eXpOS requires one user to logout before another user can login. Hence processes with multiple user id values will not execute simultaneously. Upon exit from the shell, eXpOS terminates all the descendents of the shell and awakens the login process to login the next user. The eXpOS specification allows only kernel processess (with userid 0) to execute the Login system call. The high level interface of the login system call is given here [LINK].

A special user called the root defined by eXpOS is assigned userid 1. Root is the only user with administrative priviliges.The system calls for adding a user or removing an existing user from the system can be executed only by the root (or kernel processes). A new user is created using newusr system call. An existing user can be deleted by the remusr system calls. Note that no more than eight users can be created and the operating system assigns a unique userid to each user that is created. The special users足 root or kernel cannot be removed.

A user can change his/her password using setpwd system call which takes username and password as arguments. A user, other than the root is allowed to change only his/her own password. Root can change the password of any user, including itself.

The getuid system call returns the user id of the present user. The getuname system call returns the username corresponding to a userid.

The API specification of all the multi足user system calls can be seen here

File Access Permissions

When a file is created by a process using the Create system call, the process can set its permission to exclusive to restrict access permissions. When a file is created with the exculsive flag set, it can be opened or deleted by only processes executing with the same userid using the Open/Delete system calls. The root file has defaulty userid set to 0. However root can access any file (except the root file) irrespective of the permissions. Kernel processes have unrestricted access to all system calls.
