XFS Interface Usage Specification



XFS Interface (eXperimental File System Interface) is an external interface to access the filesystem of the eXpOS. The filesystem is simulated on a binary file called disk.xfs. The interface can format the disk, load/remove files, list files and copy blocks to a UNIX file. See Filesystem specification.

Within your xfs-interface directory, use the following command to run the interface

  •       ./xfs-interface

The various commands available in XFS Interface are discussed below.

  • Help

    • Syntax : help
    • Semantics : It displays the general syntax and function of all the commands.
  • Disk Format

    • Syntax : fdisk
    • Semantics : It is used to create the disk (”disk.xfs”) or to format the disk if already created. On a newly created disk or formatted disk the Disk Free List and FAT entries are initialized.
  • Load Files

    The command load is used to load files from the UNIX filesystem to the XFS disk. The type of the file that is loaded is specified by the first argument. The second argument <pathname> is the path to the UNIX file which is to be loaded to the filesystem.
    The command checks the size of the file (executable or data files) and allocates blocks to it. A corresponding FAT entry is created for the file. For the OS startup code, interrupt routines and exception handler, the file is loaded to the corresponding location in the disk.

    The various load commands are listed below :

      1. Syntax : load --exec <pathname>
        Semantics : Loads an executable file to XFS disk.
      2. Syntax : load --init <pathname>
        Semantics : Loads INIT code to XFS disk.
      3. Syntax : load --data <pathname>
        Semantics : Loads a data file to XFS disk.
      4. Syntax : load --os <pathname>
        Semantics : Loads OS startup code to XFS disk.
      5. Syntax : load --int=timer <pathname>
        Semantics : Loads Timer Interrupt routine to XFS disk.
      6. Syntax : load --int=[4-18] <pathname>
        Semantics : Loads the specified Interrupt routine to XFS disk.
      7. Syntax : load --exhandler <pathname>
        Semantics : Loads exception handler routine to XFS disk.
  • Remove Files

    The command rm is used to remove files from the filesystem. The first argument specifies the type of file to be removed. The argument <xfs_filename> specifies the file which is to be removed.
    The command searches the FAT entries for the file (executable/data file) and clears the blocks corresponding to the file. In the case of OS startup code, interrupt routines and exception handler the corresponding blocks in the disk are cleared.

    The various load commands are listed below :

      1. Syntax : rm --exec <xfs_filename>
        Semantics : Removes an executable file from XFS disk.
      2. Syntax : rm --init
        Semantics : Removes INIT code from XFS disk.
      3. Syntax : rm --data <xfs_filename>
        Semantics : Removes a data file from XFS disk.
      4. Syntax : rm --os
        Semantics : Removes OS startup code from XFS disk.
      5. Syntax : rm --int=timer
        Semantics : Removes the Timer Interrupt routine from XFS disk.
      6. Syntax : rm --int=[4-18]
        Semantics : Removes the specified Interrupt routine from XFS disk.
      7. Syntax : rm --exhandler
        Semantics : Removes the exception handler routine from XFS disk.
  • List Files

    • Syntax : ls
    • Semantics : It lists all the files which are loaded into the filesystem. The size of the file is also displayed in number of words. The FAT entries are traversed and all the files and their corresponding sizes are displayed.
  • Display Disk Free List

    • Syntax : df
    • Semantics : It displays the disk free list. It also displays the total number of blocks and the number of free blocks.
  • Display File Contents

    • Syntax : cat <xfs_filename>
    • Semantics : It displays the contents of a file in the filesystem. The FAT entries are searched to get the blocks corresponding to the file. The blocks are displayed.
  • Copy Disk Blocks to a UNIX File

    • Syntax : copy <start_block> <end_block> <unix_filename>
    • Semantics : It copies the contents of specified block(s) in the filesystem to an external UNIX file. The arguments <start_block> and <end_block> denotes the range of blocks to be copied (including both). <unix_filename> specifes the destination UNIX file to which the contents are copied.
  • Exit Interface

    • Syntax : exit
    • Semantics : It quits the interface.
