Exec System Call

Description: used to load and run a new process in the space of a currently running process. The current process is overwritten by new process i.e. the process data structures and memory of the current process is used by the new process.

Argument: FileName

Return value: none

1. Inode Table Validation
Search for the FileName in the memory copy of Inode Table. If it doesn't exist,return with error XFS_INVFNAM. If the file entry is found in the inode table, check if the type is XEXE. If not, return with error XFS_INVFNAM. If it is valid store the Inode index of the file in InodeIndex.Also, store the code block numbers of the file in Block1 and Block2
2.Allocating Page Table Entries
Find the page table of the current process. Set the fields corresponding to code blocks with Block1 and Block2. Set the auxillary information of those fields to 00.
Check whether the new process requires use of Shared Library. If so, include the page numbers of shared library in the page table.
Check if the heap of the original process is shared by any other process. This can be done by checking memory free list. If the heap page is not shared, go to Step 4. If shared, then go to step 3
3. Allocating heap page
Find a free page in the Memory Free List and store page number in HeapPage. If not, change the state of the process to “(WAIT_MEM, _)” and call Scheduler. Repeat this Step until a free page is found.
When a free page is found, set the entry for the heap page in the Page Table to HeapPage. Set the auxillary information for the page to 01. Update the Memory Free List.
4.Post Exec Operations
Close all files opened by the current process.(Close System Call) Release all semaphores held by the parent process(Semrelease System Call)
Set SP and IP values to valid locations(link to ABI)
5. Return from system call.
