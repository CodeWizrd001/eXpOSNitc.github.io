Device Handler Interrupt

Description: It handles interrupt from I/O devices

1. Change the device status register contents to 0.
2. Check who issued the load/store instruction using Disk Status Table.
If the load/store instruction was issued by a process, go through the Process Table and wake up those processes whose state is (WAIT_DISK,_).
If the load/store instruction was issued by scheduler, then check if the instruction was a store instruction. If so, set the stored page free, by updating the memory free list and increment MEM_FREE_COUNT. If the instruction was a load, then check if the block loaded was in swap area. If so, set the block free by updating the memory copy of disk free list.
