XSM Architecture Specification

Introduction



The Experimental String Machine (XSM) is a hypothetical interrupt driven uniprocessor machine. The basic unit of data which the machine handles is the string. A string is a sequence of characters terminated by the special character '\0'. The length of a string is at most XSM_WSIZE characters including the '\0' character. (The value of XSM_WSIZE is implementation dependent). The machine treats integers, characters or sequence of characters as strings.

The basic data storage components of the machine are registers, memory, disk and the console terminal. The memory and the disk are organized as sequences of words. The word is the fundamental memory/disk storage element and each memory/disk word can store a string. In the following, we often (mis)use notation and use the term “word” to refer to the “string stored in the memory/disk word”.

  • INTERRUPTS AND EXCEPTION HANDLING

    XSM supports a total of 19 interrupts. Each interrupt is identified by its number between 0 and 18. Among these, Interrupt 0 is the exception handler, Interrupts 1, 2 and 3 are hardware interrupts. Interrupt 1 is the timer interrupt, Interrupt 2 is the disk controller interrupt and Interrupt 3 is the console interrupt. The remaining interrupts are software interrupts. Software interrupts occur when they are invoked by a program in execution. A program can invoke a software interrupt using the INT machine instruction. However, hardware interrupts and the exception handler cannot be invoked from user mode programs.

    The machine raises an exception when it runs in user mode and encounters an anomalous situation during program execution like division by zero or generation of an invalid address. All exceptions are associated with Interrupt 0. A detailed discussion on various conditions leading to the machine raising an exception are discussed in (ADD LINK).

    Associated with each interrupt/exception, there is an interrupt handler. When an interrupt occurs, the corresponding handler is executed. XSM specification reserves two contiguous memory pages for loading each interrupt handler according to the table below. When an interrupt/exception occurs, the machine transfers control to the beginning of the first page of the corresponding handler. It is the responsibility of the programmer to store the interrupt handler routines in the disk and load them to the corresponding pages at the time of system startup.

    Exceptions are anomalous situations which changes the normal flow of execution. When an exception occurs when the machine is executing in User mode, the machine transfers control of execution – pushes instruction pointer to user stack and sets the i.e. changes the value of IP to page number 2 (address = 1024) where the Exception Handler Routine resides. However in Kernel Mode, the machine halts when it encounters an exception.

    The XSM Register set consists of five special registers for storing the details regarding exceptions.

  • PAGING HARDWARE

    Paging is a memory management scheme by which a computer stores and retrieves data from the secondary storage for use in main memory. It permits the physical address space of a process to be non-contiguous. In the paging memory-management scheme, the operating system retrieves data from secondary storage in same-size blocks called pages.
