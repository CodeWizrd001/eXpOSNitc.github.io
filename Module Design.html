Kernel Module Interface



These routines are invoked from within system calls/interrupts/exception handler. Note that OS blocks only inside the modules. A module is designed to call only another module. It returns using return (RET machine) instruction.

The function number is passed by R1 and arguments are passed by registers R2, R3 etc in order. The return value (if any) is passed by R0.



Module_0 : Resource Manager


Function Number Function Name Arguments
1 Acquire Disk NIL
2 Acquire Inode Inodeindex
3 Acquire Terminal NIL
4 Release Inode Inodeindex
5 ??? ???

Module_1 : Process Manager


Function Number Function Name Arguments
1 Get Pcb Entry NIL
2 Free Pcb Entry PID
3 Exit Process PID
4 Free Page Table PID

Module_2 : Memory Manager


Function Number Function Name Arguments
1 Get Free Page NIL
2 Release Page Page Number

Module_3 : File Buffer Manager


Function Number Function Name Arguments
1 Buffer Write Disk Number, Offset, Word, Buffer Number
2 Buffer Read Disk Number, Offset, Memory Address, Buffer Number

Function Number Function Name Arguments
