System Call Interface



Application programmer interacts with the Operating System using system calls. System calls are stored in the disk and loaded into memory when the OS is loaded by bootstrap loader. When a process invokes a system call, the process is interrupted and control goes to the kernel. Therefore, system calls can also be considered as interrupts. Once the system call is carried out, the control goes back to user mode.

There are system calls associated with processes, files and semaphores. In eXpOS, there are three process system calls, nine file system calls and four system calls related to semaphores.

File System Calls

Create

Description: Create allows application programs to create new data files.

Arguments: Filename

Return Value:

0 Success
XFS_NOSP No space for file

Delete

Description: Delete is used to delete a file from the eXpFS disk.

Arguments: Filename

Return Value:

0 Success
XFS_INVFNAM File not found
XFS_FOPEN File is open

Open

Description: Open is used to open a file present in the eXpFS disk.

Arguments: Filename

Return Value:

0 - XFS_MAXFDES Success, the return value is the file descriptor for the opened file.
XFS_INVFNAM File not found or file is of invalid type
XFS_NOPSP Process has reached its limit of resources
XFS_NOFTSP System has reached its limit of open files

Close

Description: Close is used to close a file.

Arguments: File Descriptor

Return Value:

0 Success
XFS_INVFDES File Descriptor given is invalid

Read

Description: Read reads a word from a data file.

Arguments: File Descriptor

Return Value:

0 Success
XFS_INVFDES File Descriptor given is invalid
XFS_EOF File pointer has reached the end of file

Write

Description: Write writes a word into the file at the position of the file pointer.

Arguments: File Descriptor and Word to be written

Return Value:

0 Success
XFS_INVFDES File Descriptor given is invalid
XFS_NODSKSP No disk space

Seek

Description:Seek allows the application program to change the location pointed to by the file pointer so that next Read/Write is performed from the new position.

Arguments: File Descriptor and Offset specifying the number of positions by which the file pointer has to be shifted (an Offset of 0 will reset the pointer to the start of the file)

Return Value:

0 Success
XFS_INVFDES File Descriptor given is invalid
XFS_INVOFF Offset value moves the file pointer to a position outside the file

Process System Calls

Fork

Description: Replicates the process invoking the system call.

Arguments: None

Return Value:

0 - (XOS_PCOUNT - 1) Success, the return value to the parent is the process descriptor of the child process.
0 Success, the return value to the child.
XOS_PROCLIMIT Number of process has reached its maximum

Exec

Description: Loads and runs a new process in the space of a currently running process. The current process is overwritten by new process i.e. the process data structures and memory of the current process is used by the new process.

Arguments: File Name of the executable file

Return Value:

XFS_INVFNAM) File not found or file is of invalid type

Exit

Description: Destroys the process invoking the call

Arguments: None

Return Value:

-1 Failure

Getpid

Description: Returns the process identifier of the invoking process.

Arguments: None

Return Value:

Process Identifier Success
-1 Failure

Getppid

Description: Returns the process Identifier of the parent of the invoking process.

Arguments: None

Return Value:

Parent Process Identifier Success
-1 Failure

System Calls for access control and synchronization

Wait

Description: The current process is blocked till the process with PID given as argument executes a Signal system call or exits

Arguments: Identifier

Return Value:

0 Success
XOS_INVPID Given process identifier is invalid or it is the pid of the invoking process.

Signal

Description: All processes waiting for the current process are resumed.

Arguments: None

Return Value:

0 Success

FLock

Description: To lock a file so that other applications running concurrently are not permitted to access the file till the file is unlocked.

Arguments: File Descriptor

Return Value:

0 Success
XFS_INVFDES File Descriptor is invalid

FunLock

Description: Funlock operation allow an application program to unlock a file the application had locked earlier so that other applications are no longer restricted from accessing the file.

Arguments: File Descriptor

Return Value:

0 Success
XFS_INVFDES File Descriptor is invalid
XFS_NOLOCK File is already locked by some other process

Semget

Description: This system call is used to obtain a semaphore for a resource.

Argument: None

Return Value :

0- (MAXSEM-1) Success, returns the semaphore descriptor
XOS_NOPSP Process has reached its limit of resources
XOS_NOSTSP Number of semaphores has reached its maximum

Semrelease

Description: This system call is used to free the semaphore so that other processes can use it.

Arguments: Semaphore Descriptor

Return Value:

0 Success
XOS_INVSEM Semaphore Descriptor is invalid

SemLock

Description: This system call is used to lock the semaphore.

Arguments: Semaphore Descriptor

Return Value:

0 Success
XOS_INVSEM Semaphore Descriptor is invalid

SemunLock

Description: This system call is used to unlock the semaphore.

Arguments: Semaphore Descriptor

Return Value:

0 Success
XOS_INVSEM Semaphore Descriptor is invalid
