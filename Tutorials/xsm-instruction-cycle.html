XSM Instruction Execution Cycle



In this tutorial we will walk you through the instruction execution cycle of XSM architecture.
For this purpose, we will use the following SPL program that will display the contents of the first entry of the root file as the OS startup code. You can refer the SPL specification here.
The SPL program is given below: 

//Program to print the first entry of root file

alias counter R0;
alias word R1;
counter = 0;

load(62,5);

while(counter < 8) do
  word = [62 * 512 + counter];
  print(word);
  counter = counter + 1;
endwhile;


This corresponds to the following XSM code. 

// Initialising counter(R0) to 0
MOV R0, 0	

// Load block num 5 (root file) to page 62  
LOAD 62, 5

_L1:

// Check for R0 < 8
MOV R16, 8
GT R16, R0
JZ R16, _L2

// R16 = 512 * 62 + R0
MOV R16, 62
MUL R16, 512
ADD R16, R0

// Transfer contents of memory location pointed to by R16 to R1
MOV R1, [R16]

// Printing word (R1)
PORT P1, R1
OUT

// counter = counter + 1
MOV R16, R0
ADD R16, 1
MOV R0, R16

JMP _L1
_L2:

HALT

We assume that the given XSM code will be loaded into disk block 0 by the XFS Interface.
All the labels in the code are translated to corresponding machine address by the xfs-interface.

When the machine starts execution, the values of the Instruction pointer(IP) register is set to 0. The XSM machine therefore fetches the instruction present at memory location 0.
The ROM code is assumed to be present at the memory page 0. Here are the contents of the ROM code (page 0) of XSM memory 

LOAD 1,0
JMP 512

The first instruction of ROM code is to load the OS startup code from block 0 of disk to page 1 of memory.

Let us see the execution of this instruction in detail. The first step is called the Instruction Fetch (IF). The machine first the instruction from the memory and places it in the Instruction Register(IR). The second step is the Instruction decode. In this step the opcode part of the IR register is examined to see the Instruction Type.

After the execution of each instruction, the value of IP is incremented by 2 by the XSM machine. Note that each instruction in XSM occupies two words in memory.

The value of IP register is now 2. The machine fetches the instruction from memory page 2.

This instruction sets the value of IP Regiser to 512 and hence transfers control to the OS startup code loaded into page 1 of memory, ie. memory address 512.

OS startup code is loaded at memory address 512. Hence MOV R0, 0 is executed next. IP is incremented to 514. MOV R16, 62 is the instruction at memory address 514. Hence, it is executed next. This continues until IP reaches 522. The instruction at memory address 522 is JZ R16, _L2. This instruction is loaded as JZ R16, 544 into the XSM disk by the XFS Interface. This instruction changes IP according to the value of R16. If R16 is 0, IP is set to 544. Otherwise, IP increments by 2 to 524.
