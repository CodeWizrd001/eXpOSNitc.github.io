XSM Paging Hardware



The hardware paging scheme of XSM maps the virtual address space of a user mode program to the physical address space of the machine. Read the XSM Virtaul Machine Specification for an understanding of Virtual machine model provided by XSM and Virtual Address space of user programs before proceeding further.



Page Table

Every user mode program has an associated page table which maps its virtual address space to the machine's physical address space. The address of Page Table of the program currently in execution must be stored in the page table base register(PTBR) and the number of entries in this Page Table must be stored in page table length register(PTLR) for the hardware address translation to work.

The structure of a Page Table entry is shown below.

PAGE NUMBER 1
AUXILIARY INFORMATION 1
PAGE NUMBER 2
AUXILIARY INFORMATION 2
.
.

Each Page Table stores the physical page numbers corresponding to every logical page of the corresponding user mode program. The logical page numbers of a user mode program can vary from 0 to MAX_NUM_PAGES - 1. Therefore, each user mode program has MAX_NUM_PAGES entries in the page table. The current version of XSM sets MAX_NUM_PAGES to 8.

Once a page is loaded, the first word of the corresponding Page Table entry contains the physical page number where the data specified by the logical address resides in the memory.

The second word of each page table entry contains auxiliary information. The first four bits of auxiliary information are reserved as reference (R) bit, valid/invalid (V) bit, write permision (W) bit and dirty (D) bit . The remaining bits are reserved for future enhancements. The details of bits in Auxiliary information is given below.

Reference Bit (R): Initially, this bit is set to 0 (unreferenced) by the machine. On a page access, this bit is set to 1 by the machine i.e, as soon as any page is accessed for the first time, this bit is changed from 0 to 1. Page replacement algorithms of operating systems use this bit.

Valid/Invalid Bit (V): This bit indicates whether the entry of the page table is valid or invalid. The Valid/Invalid bit has value 1 if the first word of this entry corresponds to a valid physical page number. It has value 0 if the entry is invalid. The Valid/Invalid bit is set by the application, typically the operating system. If memory access is made to a page whose page table entry is invalid, the machine transfers control to the Exception Handler routine, which is responsible for loading the correct physical page.

Write Permission Bit (W): This bit is set if the user mode program is permitted to write into the page. Otherwise this bit is set to 0.

Dirty Bit (D): This bit is set to 1 by the machine if an instruction modifies the contents of the page.



Page Translation Scheme

When the machine runs in user mode, it uses logical addresses which represent locations in virtual address space of the process. In the current version of XSM, logical address can range from 0 to PTLR * 512 – 1. In the kernel mode, these logical addresses are translated to physical addresses as the kernel works only with physical addresses. In the current version of XSM, physical addresses can range from 0 to 128 * 512 – 1. The physical addresses represent locations in the primary memory of the system.

Page translation is done using the Page Table. Logical address / 512 gives the logical page number which is the index of the page in the page table. Since each page table entry has 2 words, multiplying the logical page number by 2 and adding PTBR to it gives the location of the page table entry corresponding to the given logical address.

Location of page table entry = PTBR + 2 x (Logical Address / 512)

The value stored in the first word in the page table entry corresponds to the physical page number.

Physical Page Number = [ Location of page table entry ]

Offset into the page is calculated as

offset = Logical Address % 512

The physical address is computed by multiplying the physical page number by page size (=512) and adding the offset.

Physical Address = Physical Page Number x 512 + offset





NOTE: In user mode, logical addresses are translated to physical address by the machine using the address translation scheme. However, in kernel mode, the translation must be done manually using the procedure described above.

Example:

Assume the per-process page table is as shown below:



Physical Page Number Auxiliary Information
19 01
20 01
-1 00
-1 00
57 11
48 11
72 11
-1 00

Given a logical address of 3532, the translation occurs as follows:

Logical page number = index of page table entry

= 3532/512 = 6

Location of page table entry = PTBR + 2 * 5

Physical Page Number = [PTBR + 2 * 5 ] = first word of the 6th entry in page table = 48

Offset = 3532%512 = 460

Physical Address = 48 * 512 + 460 = 25036
